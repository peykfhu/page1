API 文档

AnPin AI 兼容 Anthropic / OpenAI API 格式，可无缝切换使用。

简介

AnPin AI 提供统一的 API 网关服务，通过一个 API Key 即可访问 Claude、GPT、Gemini、DeepSeek 等多种大模型。API 接口与 Anthropic / OpenAI 官方完全兼容，现有代码仅需修改 Base URL 即可接入。

💡 如果你已经在使用 Anthropic 或 OpenAI 的官方 SDK，只需更换 base_url 和 api_key 即可立即使用 AnPin AI。

认证方式

所有 API 请求都需要在 HTTP Header 中携带 API Key 进行认证。支持以下两种格式：

Anthropic 格式

x-api-key: sk-anpin-xxxxxxxxxxxx

Bearer Token 格式

Authorization: Bearer sk-anpin-xxxxxxxxxxxx
⚠️ 请勿在客户端或前端代码中暴露 API Key。API Key 应仅在服务端使用。

Base URL

将请求的 Base URL 替换为你的 AnPin AI 部署地址即可：

# Anthropic 兼容端点
https://your-domain.com

# OpenAI 兼容端点
https://your-domain.com/openai

Messages API

发送消息并获取 AI 回复，兼容 Anthropic Messages API。

请求

项目值
方法POST
路径/v1/messages
Content-Typeapplication/json

请求体参数

参数类型必填说明
modelstring是模型名称，如 claude-sonnet-4-20250514
messagesarray是对话消息列表
max_tokensinteger是最大输出 Token 数
systemstring否系统提示词
temperaturenumber否采样温度 (0-1)，默认 1
top_pnumber否Top-P 采样 (0-1)
top_kinteger否Top-K 采样
streamboolean否是否启用流式输出
stop_sequencesarray否停止生成的字符串序列
metadataobject否请求元数据，如 user_id

示例请求

curl https://your-domain.com/v1/messages \
  -H "Content-Type: application/json" \
  -H "x-api-key: sk-anpin-xxxx" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "claude-sonnet-4-20250514",
    "max_tokens": 1024,
    "system": "你是一个专业的技术顾问。",
    "messages": [
      {"role": "user", "content": "什么是 API 网关？"}
    ]
  }'

示例响应

{
  "id": "msg_xxxx",
  "type": "message",
  "role": "assistant",
  "content": [
    {
      "type": "text",
      "text": "API 网关是..."
    }
  ],
  "model": "claude-sonnet-4-20250514",
  "stop_reason": "end_turn",
  "usage": {
    "input_tokens": 25,
    "output_tokens": 150
  }
}

Streaming（流式输出）

在请求中设置 "stream": true 即可启用 Server-Sent Events (SSE) 流式输出，实时获取模型生成的内容。

请求示例

curl https://your-domain.com/v1/messages \
  -H "Content-Type: application/json" \
  -H "x-api-key: sk-anpin-xxxx" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "claude-sonnet-4-20250514",
    "max_tokens": 1024,
    "stream": true,
    "messages": [
      {"role": "user", "content": "写一首短诗"}
    ]
  }'

SSE 事件类型

事件说明
message_start消息开始，包含消息元数据
content_block_start内容块开始
content_block_delta内容增量更新（文本片段）
content_block_stop内容块结束
message_delta消息级别更新（stop_reason 等）
message_stop消息完成

OpenAI 兼容接口

AnPin AI 同时提供 OpenAI Chat Completions 兼容接口，方便已有 OpenAI 集成的项目迁移。

项目值
方法POST
路径/openai/v1/chat/completions

示例请求

curl https://your-domain.com/openai/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-anpin-xxxx" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "Hello!"}
    ]
  }'

Python (OpenAI SDK) 示例

from openai import OpenAI

client = OpenAI(
    api_key="sk-anpin-xxxx",
    base_url="https://your-domain.com/openai/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "user", "content": "你好"}
    ]
)
print(response.choices[0].message.content)

多模态 / 视觉

支持发送图片给模型进行视觉理解。在 messages 中使用 image 类型的 content block：

{
  "model": "claude-sonnet-4-20250514",
  "max_tokens": 1024,
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "image",
          "source": {
            "type": "base64",
            "media_type": "image/png",
            "data": "iVBORw0KGgo..."
          }
        },
        {
          "type": "text",
          "text": "请描述这张图片"
        }
      ]
    }
  ]
}
💡 支持的图片格式：JPEG、PNG、GIF、WebP。单张图片最大 5MB。

错误处理

API 返回标准 HTTP 状态码。出错时，响应体包含 JSON 格式的错误信息：

{
  "type": "error",
  "error": {
    "type": "invalid_request_error",
    "message": "max_tokens: Field required"
  }
}
状态码类型说明
400invalid_request_error请求参数错误
401authentication_errorAPI Key 无效或未提供
403permission_error权限不足或账户已被禁用
404not_found_error请求的模型或资源不存在
429rate_limit_error请求频率超限
500api_error服务器内部错误
529overloaded_error上游过载，请稍后重试

速率限制

速率限制取决于你的账户等级和套餐配置。超出限制时会返回 429 状态码。响应头中包含限制信息：

Header说明
x-ratelimit-limit时间窗口内允许的最大请求数
x-ratelimit-remaining当前窗口剩余请求数
x-ratelimit-reset限制重置的时间戳

建议在代码中实现指数退避重试机制：

import time
import anthropic

client = anthropic.Anthropic(
    api_key="sk-anpin-xxxx",
    base_url="https://your-domain.com"
)

for attempt in range(3):
    try:
        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=1024,
            messages=[{"role": "user", "content": "Hello"}]
        )
        break
    except anthropic.RateLimitError:
        time.sleep(2 ** attempt)
